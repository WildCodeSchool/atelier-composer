Installer code sniffer sur un projet

Code sniffer est un outil (écrit en PHP) permettant de détecter les portions de code ne respectant pas les coding style définis. Il gère différentes règles dont les PSR-1 et PSR-2

Comment je l'installe ? Avec composer !

  • Reprendre l’atelier POO (sur les chats),
  • Installer composer (s'il n'est pas déjà installé globalement sur ta machine)
  • Initialiser le projet (création du composer.json)
  • Chercher le paquet squizlabs/php_codesniffer sur packagist
  • L’installer via composer

Utiliser Code Sniffer

Code Sniffer s’installe dans le dossicer vendor/. Il se lance en ligne de commande (rappelle toi, un script PHP peut aussi être lancé en mode CLI ;-)) : 

php vendor/bin/phpcs myFile.php

De base il ne respecte pas les PSRs (mais d’autres règles). Il faut lui spécifier en option : 

php vendor/bin/phpcs --standard=PSR2 myFile.php

Corrige les éventuelles erreurs détectées par code sniffer sur les différents fichiers du projet.

Et voilà ! Grâce à composer, tu as installé en un clin d'oeil code sniffer. Et grâce à ce dernier, tu as corrigé efficacement tes erreurs de coding style, et ton code est tout beau, tout propre !
